# ADR-0026 — Significance-band отбора: paired-bootstrap по OOF, честный выбор по умолчанию

- **Статус:** Accepted (реализован 2026-06-08, M4a1 ядро + M4a2 дефолт-ON/наблюдаемость; impl-notes в §2/§4)
- **Дата:** 2026-06-08
- **Драйверы:** DM4-1 (честный выбор по умолчанию); FR-M4-1/2/3/4, NFR-M4-1/2/7/8. Наследует
  ADR-0007 (контракт абсолютного отбора + band) и ADR-0022 (изоляция/OOF). Питается SPIKE-0002.
- **Воркстрим:** M4a. Активирует уже разведённый, но инертный `select_best`-band.

## Контекст
`select_best` (`selection_policy.py:64-93`) уже строит equivalence-band вокруг абсолютного
argmax-anchor и делает лексикографический тай-брейк (компактность→стабильность→скорость), но
под `NoSignificanceTest` (`facade.py:80`) band всегда `[anchor]` ⇒ чистый argmax. Ни один аналог
(FLAML/AutoGluon/MLJAR/LAMA) не даёт band — это дифференциатор. SPIKE-0002 подтвердил метод и вскрыл
нюанс: post-hoc выбор anchor (argmax) делает one-sided тест либеральным (Type-I ~6–10%) ⇒ членство
консервативно (two-sided). **Честная формулировка claim (фикс R2-major):** гарантия — «победитель **не
значимо хуже** абсолютно-лучшего (при контроле ложного исключения), а среди статистически-неразличимых
выбран **простейший/быстрейший** (Occam)», а НЕ буквально «лучший по метрике». Это и есть честность:
не объявлять различие, которого нет. two-sided консервативен (Type-I 1.000, power умеренной разницы
0.88) — band может быть непуст и победителя решает тай-брейк; чтобы это было ВИДНО (а не «декоративно»),
наблюдаемость несёт ширину band и признак «выбран тай-брейком» (§6, NFR-M4-7). `alpha` — настраиваемый
рычаг; one-sided-с-margin — задокументированная альтернатива (future), дефолт — консервативный two-sided.

## Рассмотренные варианты
1. **Per-sample paired bootstrap по фикс. loss** (log-loss/0-1). Просто, но **не работает для
   ранжирующих метрик** (roc_auc не раскладывается в сумму по-сэмпловых потерь). Отвергнут.
2. **Paired bootstrap, рекомпьют МЕТРИКИ на ресэмпле** (anchor vs кандидат по одним индексам);
   членство по two-sided CI-overlap разности. Универсально (auc/log_loss/accuracy/rmse), SPIKE-0002
   подтвердил для proba/0-1 И **для ранжирующего roc_auc** (Type-I 1.000, power явный 1.0 / умеренный
   0.88 на two-sided) и дешевизну. **Выбран.**
3. Naive k-fold-mean paired t-тест **и** Nadeau-Bengio corrected resampled t-тест. Fold-mean t —
   малостепенной/некорректный (Dietterich 1998). Nadeau-Bengio — **отвергнут** в пользу bootstrap:
   непараметрик проще, работает для ранжирующих метрик, не требует нормальности; corrected-t дал бы
   тот же band ценой допущений. Отвергнуты оба.

## Решение

### 1. Адаптер `BootstrapSignificanceTest` (реализует существующий порт)
`adapters/significance.py`, реализует `SignificanceTest`: поля `seed:int`, `n_boot:int`, `metric:
Metric`. **Порт расширяется АДДИТИВНО (фикс R2-blocker B1):** `equivalent(pred_a, pred_b, y_true, *,
alpha, block_index=None, sample_weight=None) -> bool` — `block_index`/`sample_weight` НЕ поля адаптера,
а аргументы, которые **`equivalence_band` (§6) передаёт уже ВЫРОВНЕННЫМИ к плотному срезу** (`block_index[mask]`,
`sample_weight[mask]`), т.к. векторы тоже приходят срезанными по единой маске. Это устраняет
рассинхрон «вектор в одном пространстве индексации, блок-структура в другом».
- **Порядок аргументов (фикс R1):** `equivalence_band` зовёт `equivalent(cand[mask], anchor[mask],
  y[mask], ...)` ⇒ **`pred_a` = кандидат, `pred_b` = anchor**. Знак не важен: `Δ = metric.score(rs,
  pred_b) − metric.score(rs, pred_a)`, членство по **two-sided** CI «включает 0» — инвариантно к
  порядку и `greater_is_better`.
- ресэмпл строк **с возвратом**, ОДНИ индексы для обеих моделей (paired); `n_boot` раз; на каждом —
  `metric.score(y_true[idx], pred[idx], sample_weight[idx])` для обеих ⇒ распределение `Δ`. **Вес
  передаётся в score на каждом ресэмпле** (иначе band ≠ взвешенному критерию отбора — фикс R2).
- **членство (two-sided CI-overlap, фикс SPIKE-0002):** кандидат **в band**, если **two-sided
  `(1−alpha)` CI разности `Δ` включает 0** (не `p>alpha` на границе). SPIKE-0002 (доп. прогон):
  Type-I=1.000 (идеально консервативно, устраняет post-hoc-anchor либеральность one-sided 0.92);
  power: явная разница 1.0, умеренная 0.88 (честное направление — near-tie в band).
- **Ранжирующие метрики (roc_auc) — валидны (фикс R1-adv):** вариант 1 (per-sample loss) отвергнут
  именно из-за roc_auc, но выбранный рекомпьют МЕТРИКИ на ресэмпле для AUC **проверен** SPIKE-0002:
  дубликаты ресэмпла затрагивают обе парные модели одинаково и **сокращаются в разности** `Δ`.
- **Единая band-маска — НЕ циклически (фикс R2-blocker B2):** маска вычисляется **ОДИН раз ДО теста**
  как пересечение `_valid_mask` по **фиксированному набору = все не-провалившиеся кандидаты с непустой
  OOF** (провалившиеся уже отсеяны ADR-0022). Затем band выбирается тестом НА этой фиксированной маске:
  `mask → test → band` (нет цикла «маска зависит от band»). Anchor и каждый кандидат пересчитываются на
  ОДНОМ множестве строк (для неаддитивной auc `Δ` сопоставимы). **Компромисс:** при стандартной k-fold
  OOF покрытие полное у всех ⇒ маска = все строки; при частичной holdout-OOF (ADR-0010 §7) разреженный
  кандидат сужает n у всех — документируется; при необходимости пересечение ограничивается top-K по
  рангу (future). Владелец маски/выравнивания — core-функция `equivalence_band` (§6), а не адаптер.
- **Multiclass-форма (фикс R1-adv):** `oof_pred` для multiclass — `(n,K)` (§3); `_valid_mask`
  считается **построчно** (`~np.isnan(pred).any(axis=1)` для 2-D), индексация по строкам, `metric.score`
  на `(m,K)`. `equivalent` не предполагает 1-D.

### 2. Time-series → FOLD-level block bootstrap (фикс R1-consistency + R2-blocker B1)
i.i.d. bootstrap занижает дисперсию при автокорреляции (SPIKE-0002), НО OOF идёт в **порядке строк**
(ADR-0027 §1 не переупорядочивает) — moving-block по нему неверен. **Решение:** для TS ресэмплируются
**целые CV-тестовые фолды** (смежные временные блоки): `block_index` = номер фолда каждой OOF-строки.
**Выравнивание (фикс B1):** `block_index` строится composition из сплиттера в порядке ИСХОДНЫХ строк
(длина n); `equivalence_band` (§6) сама делает срез `block_index[mask]` и передаёт его в `equivalent`
**вместе с уже срезанными** `cand[mask]`/`anchor[mask]`/`y[mask]` — все в одном пространстве индексации
плотного среза. Адаптер ресэмплирует фолды (по значениям `block_index[mask]`) с возвратом, метрика — на
объединении выбранных. Для не-TS `block_index=None` ⇒ i.i.d. row-bootstrap.
**Детерминизм (фикс R1-repro):** seed каждой пары — из **СТАБИЛЬНОГО ключа** (`SeedSequence(seed)`
+ hash отсортированного `candidate.id`, НЕ позиция в ranked-списке) ⇒ band инвариантен к добавлению
заведомо-худшего кандидата (ADR-0007).
> **Реализация M4a1:** порт `equivalent` не несёт `candidate.id`, поэтому seed реализован проще и
> сильнее — `np.random.default_rng(self.seed)`, **сбрасываемый на каждый вызов** `equivalent`. Каждая
> пара тянет одни и те же ресэмплы (общие случайные числа), независимо от состава/порядка пула ⇒
> band инвариантен к добавлению кандидата (NFR-M4-2, тест `test_band_invariant_to_worse_candidate`).
> Это инвариантность строже hash(id) (не зависит ни от ранга, ни от id) и устраняет необходимость
> протаскивать id через порт. Цель §2 достигнута; механизм — иной.

### 3. OOF для не-proba + смена семантики `oof_pred` (FR-M4-2; фикс R1)
`_run_candidate` расширяется: `oof_pred` хранит **метрика-готовый OOF-вектор** для ВСЕХ метрик —
proba `(n)`/`(n,K)` (как сейчас), либо предсказанный класс (class-метрики), либо значение
(value-метрики: RMSE/MAE). Форсинг захвата — в `run_slice:209` (`need_proba`-логика расширяется на
class/value при активном тесте), `_wants_oof` остаётся чистым предикатом. **Это смена СЕМАНТИКИ
существующего поля** (`oof_pred` был строго `P(positive)`), не чисто аддитив (фикс R1): (а) валидность
строк определяется **`oof_mask`** (а НЕ `np.isnan` — для int/str class-векторов `~np.isnan` падает);
(б) для калибровки (ADR-0030) нужен ОТДЕЛЬНЫЙ proba-канал `oof_proba`, не этот метрика-готовый вектор.
**§3 — предусловие §4** (фикс R1): расширение захвата OOF реализуется/мёржится ДО включения дефолт-ON,
иначе регрессионный/accuracy-прогон тихо теряет band.
> **Расширено ADR-0031:** при `selection="refinement"` (opt-in) для proper-proba-метрик `oof_pred`/
> `score` кандидата = **cross-fitted калиброванные** (refinement, Berta et al. 2025), а не сырые; band
> остаётся метрика-агностичной (работает на подменённом `oof_pred`). Дефолт `raw` — без изменений.

### 4. Дефолт ON (контракт-change исхода отбора; после §3)
Composition/`facade` передаёт `BootstrapSignificanceTest(metric, seed, n_boot=...)` вместо
`NoSignificanceTest()`. Band — поведение **по умолчанию** (north-star). Отключение — программно явным
inert-тестом (публичный конфиг-тумблер → M5; фикс R1). Прогоны с однозначным победителем не меняют
исход (band=[anchor]); near-tie — победитель по тай-брейку + **instability-flag** (§5). Контракт-change
vs чистый argmax (lineage ADR-0007). **Порядок реализации внутри M4a:** §3 (OOF-захват) → §6 (core
band-функция) → §4 (дефолт-ON).
> **Реализация M4a2 (n_boot, обоснованное уточнение):** ADR изначально фиксировал `n_boot=2000`. SPIKE
> мерил `~0.1–0.2 с` для **векторизованного** ресэмпла; реализация считает метрику per-resample через
> порт `Metric` (sklearn-вызов на ресэмпл, без метрика-специфичного быстрого пути — ради чистоты
> `core`), поэтому стоимость = `n_boot` sklearn-вызовов на пару (постоянный множитель на fit, NFR-M4-8
> соблюдён — не nested). Измерено: 8000 `roc_auc_score` ≈ 5.2 с. Чтобы ограничить эту константу,
> **производственный дефолт снижен до `n_boot=1000`** — SPIKE прямо допускает («≥1000 достаточно,
> flip-rate 0%»; 2000 было запасом) и это ровно порог §7 (`1000·0.05=50`). `2000` доступно явной
> передачей. Векторизованный быстрый scorer для band — future-оптимизация, если понадобится.

### 5. Instability-flag; stability — ВНЕ M4 (фикс R2-major)
Флаг — через core-функцию §6: band с anchor=top-1 и top-2, при различии состава `band_unstable=True`
(+ при ПОЛНОЙ ничье, когда выбор максимально произволен, §7). Ограничение top-k-кластера документируется.
**`Candidate.stability` per-fold НЕ заполняется в M4** (фикс R2): это потребовало бы per-fold scoring в
`_run_candidate` (K× `metric.score`/кандидат — реальная стоимость + семантически шумный per-fold AUC,
конфликт с pooled-OOF выбором ADR-0027 §4) — это **не «аддитив», а расширение обязанностей use-case**.
Поэтому per-fold scoring + populяция `stability` **выносятся в M7** (вместе с ES-обучением, где per-fold
данные и так появятся). В M4 тай-брейк = **`n_features`→`train_time`** (компактность→скорость, Occam);
`stability` остаётся `0.0` (no-op, явно). Это убирает риск горячего OOF-пути и скрытую развилку
воспроизводимости между подэтапами.

### 6. Core-функция `equivalence_band` (владеет маской/выравниванием) + наблюдаемость
Вводится **чистая core-функция** `equivalence_band(candidates, policy, test, y_true, *,
block_index=None, sample_weight=None) -> BandResult(member_ids, winner, unstable)`
(`core/selection_policy.py`, Humble Object — numpy, без I/O). **Она ВЛАДЕЕТ всей выровненной логикой
(фикс B1/B2):** (1) строит фиксированную единую маску (§1); (2) для каждого кандидата зовёт
`test.equivalent(cand[mask], anchor[mask], y[mask], alpha, block_index=block_index[mask],
sample_weight=sample_weight[mask])` — все срезы согласованы; (3) собирает band, тай-брейк, top-1/top-2
instability. `select_best` делегирует ей и возвращает `winner` (попарная `both`-маска `selection_policy.py:85`
ЗАМЕНЯЕТСЯ единой). **Контейнер наблюдаемости (фикс R2-day2 forward-compat):** результаты честности —
`band_member_ids`, `band_unstable`, **`band_width`** (число членов) и **`winner_by_tiebreak: bool`**
(победитель = anchor или выбран тай-брейком — NFR-M4-7) — кладутся в `SliceResult` (in-memory) и в
**artifact-манифест** (plain dict + `.get`-fallback, **симметрично forward/backward**, как holdout/
calibration). **`LeaderboardEntry` (frozen, extra=forbid) НЕ трогается** — `in_band` НЕ добавляется в
него (иначе старая сборка не прочитает новый `leaderboard.json`, фикс R2-day2); членство band —
ключ манифеста (`band_member_ids`), не поле frozen-entry.
**Публичная поверхность фасада (фикс R2-completeness):** `AutoML.fit` публикует sklearn-конвенциональные
`*_`-атрибуты: `band_member_ids_`, `band_unstable_`, `holdout_score_` (None при `outer_holdout=0`),
`calibration_` (метод/applied/Brier-ECE до-после, None при off), `reliability_curve_` (None при off) —
проводка `SliceResult → self.*_` в `facade.fit` (имена атрибутов = публичный контракт, фиксируется сейчас).

### 7. Граничные контракты (фикс R2-completeness)
- **Пустая/малая единая маска:** если общая маска даёт `< min_rows` строк (дефолт напр. `2*n_classes`),
  кандидат **консервативно ВКЛЮЧАЕТСЯ в band** (не звать `metric.score` на вырожденном срезе → не
  ловить sklearn-`ValueError` внутри bootstrap). Порог документируется.
- **`n_boot` vs `alpha`:** требовать `n_boot * alpha >= 50` (хвост перцентильного CI осмыслен), иначе
  `ConfigError` на границе (не молчаливый мусорный CI). Дефолт `n_boot=2000`, `alpha=0.05` → 100, ок.
- **Полная ничья (все в band):** `band_unstable=True` (выбор по тай-брейку максимально произволен —
  наблюдаемость нужнее всего); победитель — тай-брейк (детерминирован).
- **Вырожденное bootstrap-распределение** (нулевая дисперсия `Δ`): включить (operational §5).

## Последствия
- **Положительные:** активируется дифференциатор; универсально по метрикам (вкл. roc_auc, измерено);
  дёшево (постоянный множитель, NFR-M4-8); детерминированно (seed по стабильному id); band/флаг
  наблюдаемы через явный core-контейнер.
- **Отрицательные/компромиссы:** дефолт-ON меняет исход отбора на near-tie (контракт-change,
  программный откат до M5); захват OOF всегда (доп. проход) — оправдано честностью; two-sided
  консервативен (умеренная разница: мощность 0.88, может оставить чуть-худшего в band) — компенсируется
  Occam-тай-брейком (простейший из неразличимых) + наблюдаемостью «выбран тай-брейком»; per-fold
  `stability` отложена в M7.
- **Влияние на слои (ядро правится АДДИТИВНО):** новый адаптер `adapters/significance.py` (реализует
  core-порт `SignificanceTest`, расширенный **аддитивными** kwargs `equivalent(..., block_index=None,
  sample_weight=None)`); **аддитивные правки `core`:** чистая функция `equivalence_band` + `BandResult`
  (владеет маской/выравниванием); **`LeaderboardEntry` НЕ трогается** (in_band — ключ манифеста);
  расширение OOF-захвата — `application/slice.py`; проводка + публичные `*_`-атрибуты — `composition`.
  import-linter 3/3 KEPT. numpy-only в значимости (scipy опц. — NFR-M4-4/6).

## Проверки
- Type-I: две эквивалентные модели → один band ≥0.95 — **измерено SPIKE-0002**: roc_auc two-sided
  **1.000**, log_loss/0-1 two-sided ≥0.95; тест на синтетике.
- Power: явно-худшая исключена (**измерено**: roc_auc two-sided **1.0** явная, 0.88 умеренная).
- Ранжирующая метрика: band на roc_auc (дефолт binary) корректен — тест Type-I/power на AUC.
- Общая band-маска: кандидаты с РАЗНЫМИ OOF-масками дают согласованный band (anchor на единой маске).
- Multiclass: band на `(n,K)`-proba (`_valid_mask` построчно) — тест.
- Не-proba: band активен на регрессии (RMSE) и accuracy; маска не на `np.isnan` для class/value (FR-M4-2).
- Детерминизм: фикс `seed` ⇒ идентичные band/победитель; **band инвариантен к добавлению заведомо-
  худшего кандидата** (seed по id, NFR-M4-2).
- TS: fold-block bootstrap даёт шире CI, чем i.i.d. (SPIKE-0002 fold/iid ratio >1); тест.
- Instability-flag на near-tie и полной ничье, не на отрыве (FR-M4-3); `band_member_ids`/`band_width`/
  `winner_by_tiebreak` в манифесте; `leaderboard.json` НЕ меняется (forward/backward symmetric, NFR-M4-5).
- Граничные: пустая маска → включить; `n_boot*alpha<50` → `ConfigError`; weighted resample (§7).
- Публичные `*_`-атрибуты фасада присутствуют после fit (`band_member_ids_` и т.д.).
- Дефолт-ON: однозначный победитель не меняется; near-tie → тай-брейк + флаг (FR-M4-4).
- Чистота: `equivalence_band`/`equivalent` тестируются на numpy без обучения (NFR-M4-6).
