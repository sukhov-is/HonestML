# ADR-0004 — Тулинг, типизация, CI и версионирование

- **Статус:** Proposed
- **Драйверы:** D6; NFR-2, NFR-3, NFR-8, NFR-9, NFR-12.
- **Воркстрим:** M0.

## Контекст

Нет единого стека качества: нет линта/форматтера/типчекера/CI, тесты только
smoke, нет `py.typed`, нет политики версий. Для OSS со «стабильным API» и
правилом слоёв (NFR-2) это обязательная база.

## Рассмотренные варианты

- Линт/формат: **ruff** (быстрый, lint+format в одном) vs flake8+black+isort
  (несколько инструментов). → ruff.
- Типы: **mypy** (зрелый, строгий) vs pyright. → mypy (строго на `core/`,
  мягче на адаптерах из-за внешних типов).
- Правило слоёв: **import-linter** (контракты слоёв) — фиксирует, что домен не
  импортирует инфраструктуру (NFR-2).
- Тесты: pytest + страты (unit/property/golden/slow); property — hypothesis.

## Решение

- **ruff** (lint + format), конфиг в `pyproject.toml`; **mypy** строгий на
  `src/honestml/core/**`, базовый на адаптерах; **import-linter** с контрактом
  «`core` ← `application` ← `adapters` ← `composition`, домен ни на кого наружу».
- **`py.typed`** маркер (типы экспортируются потребителю).
- **pytest** + маркеры: `unit` (чистое ядро, по умолчанию), `property`
  (hypothesis: OOF без утечки, монотонность бюджета), `golden`
  (детерминизм метрик на синтетике, seed), `slow` (полный прогон, отдельный job).
- **pre-commit** (ruff/mypy/end-of-file/trailing-whitespace).
- **Поддержка Python:** 3.10+ (современный typing; в коде уже
  `from __future__ import annotations`). Матрица CI: ОС {linux, windows, macos} ×
  py {3.10, 3.11, 3.12}; job'ы: lint+type, unit+property, golden, slow (только
  linux/latest), docs-build.
- **Версионирование:** SemVar; политика депрекации — предупреждение ≥1 минор до
  удаления; публичная поверхность API фиксируется тестом (что экспортируется).
- **Docs:** mkdocs-material (собирается в CI); quickstart + API + контракт плагина
  наполняются в M9, каркас — здесь.

## Последствия

- (+) Качество и правило слоёв проверяются автоматически с первого воркстрима.
- (+) `import-linter` делает Clean Architecture исполняемым инвариантом, а не
  пожеланием.
- (−) Начальная настройка CI-матрицы и строгого mypy на ядре; внешние библиотеки
  без типов потребуют точечных `# type: ignore`/stub-ов (ограничиваем адаптерами).
- Точные пороги (покрытие ядра, допуск бюджета) фиксируются как `(TBC)` в NFR и
  уточняются по мере наполнения.
