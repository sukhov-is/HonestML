# ADR-0048 — Locus честного арбитража: DEV-внутренний selection-holdout; внешний holdout остаётся нетронут

- **Статус:** Accepted (реализован в M6c, 2026-06-10; информирован **SPIKE-M6c-1**). **Impl-notes:** carve
  переиспользует `outer_holdout_carve` (DRY, не отдельная функция); `embargo` к одиночному carve неприменим →
  **purge-only** (паритет с внешним holdout, §3); арбитр выравнивает proba по whole-DEV `global_classes`
  (multiclass-фикс ревью). См. `09-review.md`.
- **Дата:** 2026-06-10
- **Драйверы:** DM-C1 (честность арбитража — headline). FR-FSC-4, NFR-FSC-1. Наследует honest-regime holdout
  (ADR-0029) и его инвариант «holdout не участвует в отборе»; scheme-aware carve (`outer_holdout_carve`).
- **Воркстрим:** M6c.

## Контекст
honest-compare выбирает **один** subset-победитель из N. Выбор лучшего по метрике **смещает** ту выборку, на
которой судит (winner's curse). **Где судить?** Три кандидата: (1) внешний holdout ADR-0029; (2) DEV-OOF (выбор =
inner-CV-knob); (3) DEV-внутренний selection-holdout (nested). Два осложнения: **(а)** нельзя пачкать
несмещённость внешнего holdout; **(б)** среди стратегий есть **wrapper** `sequential`, который **сам
оптимизирует** OOF-метрику (ADR-0047 §2) → арбитраж на DEV-OOF дал бы ему «домашнее преимущество» (нечестно
против ранжирующих). Нужна **независимая** от внутреннего отбора каждой стратегии поверхность.

## Данные SPIKE-M6c-1 (250 повторов, ROC-AUC)
- best-of-N смещение мало для N=2–3 (**~0.002–0.003 AUC mean**), доминируется шумом размера holdout; regret
  ≤0.0015 (арбитр надёжно берёт near-best).
- **Ключевой принцип (Эксп. C):** отбор, **не касающийся** внешнего holdout, оставляет его **несмещённым**
  (bias ≈ **−0.0003**); прямой выбор на внешнем holdout — смещение **+0.0013** (малое, но **избегаемое**).
- Малая selection-выборка зашумляет арбитраж (Эксп. B: 150 → +0.005; ≥300 → <0.0015).
- **Граница применимости чисел (фикс R1-A6, adversarial):** магнитуды (~0.002–0.003) получены на **синтетике**
  (50 признаков, 8 информ., `flip_y=0.05`, ROC-AUC) и на **прокси** «N случайных subset'ов равного размера»
  (worst-case по winner's curse), **не** на реальных learned-subset'ах. → это **иллюстративная оценка порядка**,
  не универсальная гарантия: другие метрики (log-loss/F1/RMSE) и реальные данные могут давать иные магнитуды.
  **Что робастно и определяет решение — структурный вывод Эксп. C** (отбор вне внешнего holdout ⇒ внешний holdout
  несмещён); он **не** зависит от магнитуды. Локус выбран по структуре, не по числу. (Эмпирическая валидация на
  реальных данных — Day-2/M6d.)

## Рассмотренные варианты
1. **Арбитраж на внешнем holdout (ADR-0029).** Просто, но **ломает** инвариант «holdout вне отбора» → holdout
   становится критерием выбора; смещение **избегаемо** (spike C). Для timeseries усиливается дрейфом
   (R-FSC-TS-DRIFT). **Отвергнут** (пачкает honesty-якорь без нужды).
2. **Арбитраж на DEV-OOF (reuse фолдов).** Дёшево, использует все DEV, внешний holdout нетронут. **Но
   нечестно** к ранжирующим стратегиям: `sequential` оптимизировал ровно эту OOF-метрику → системный перекос к
   wrapper. **Отвергнут** (несправедливый арбитр подрывает «честно лучший»).
3. **DEV-внутренний selection-holdout (nested), независимый от внутреннего отбора стратегий.** Внешний holdout
   нетронут (spike C ≈ 0); поверхность **не оптимизировалась** ни одной стратегией → **честное сравнение**
   разнородных методов. Цена — DEV-данные + шум на малых выборках. **Выбран.**

## Решение

### 1. Дефолт: арбитраж на DEV-внутреннем selection-holdout
Только в **compare-режиме** (N>1):
1. Из **DEV** scheme-aware вырезается `selection-holdout` → `dev_fit` + `sel_holdout`. **Carve — инъекция, не
   прямой import адаптера** (фикс R1-A1): `composition` резолвит callable, **переиспользуя существующий
   `outer_holdout_carve(dev_ds, scheme, fraction=selection_holdout, stratify, random_state, purge)`** (DRY —
   тот же scheme-механизм: timeseries=future+purge, group=disjoint, иначе stratified), и передаёт его в
   `application`-арбитр — **как `splitter` инъектируется в `run_slice` (ADR-0016)**. `application` зависит от
   **сигнатуры** carve, не от адаптера → `layers`/`usecases-independent-of-adapters` KEPT.
2. Каждая стратегия выбирает subset **только на `dev_fit`** (анти-ликедж per-strategy сохранён; `sel_holdout`
   **не участвует** во внутреннем отборе ни одной стратегии).
3. **Арбитр** фитит estimator-agnostic рэнкер-модель (ExtraTrees, как M6b) на `dev_fit[:, subset_i]`, скорит на
   `sel_holdout[:, subset_i]` целевой метрикой → `arb_score_i`. Победитель = `argmax`. **Tie-break (R1-A12):** при
   равных скорах (в т.ч. совпадающих subset'ах) — **первая** стратегия по порядку `compare` (детерминировано);
   `run_report` перечисляет все N без dedup. Поверхность **независима** от внутреннего отбора любой стратегии.
4. Subset-победитель **замораживается**; `sel_holdout` **возвращается** в DEV: лидерборд кандидатов и `refit_best`
   идут на **полном DEV** с замороженным subset (как M6b). **Фолды лидерборда/refit — заново строятся на full-DEV**
   тем же splitter'ом, что в M6b (фикс R1-A5): carve-разбиение `dev_fit/sel_holdout` — **временное устройство фазы
   отбора стратегии**, оно **отбрасывается** до лидерборда; фолды оценки **не наследуют** арбитражное разбиение →
   арбитражная информация не протекает в leaderboard-скор.
5. Внешний holdout (ADR-0029) **нетронут** → один раз скорит финальную модель → **несмещённая** оценка всего
   пайплайна (вкл. выбор стратегии). Spike C это подтверждает.

### 2. Граница честности (документируется, без overclaim — NFR-FSC-1)
- Внешний holdout остаётся **независимой несмещённой** проверкой (никогда не участвует в выборе стратегии/модели/
  subset). honest-compare смещает **selection-holdout** (на нём судим), но **не** внешний holdout.
- **Что именно даёт независимость (уточнено R1, adversarial):** `sel_holdout` — **отдельная оценочная поверхность
  с иной фолд-структурой**, чем внутренние циклы стратегий (она не входит ни в `dev_fit`-ранжирование ranker'ов,
  ни в OOF-скоринг `sequential` на `dev_fit`-фолдах). Это **metric-independent surface** (другие строки), **не**
  «метрика-ортогональность»: `sequential` всё ещё оптимизирует CV-прокси на `dev_fit`, поэтому остаточный
  structural-перекос к wrapper **смягчён** (оценка на disjoint-строках), но **не устранён** полностью — снят
  внешним holdout. Честно, без overclaim.
- Остаточный **inner-оптимизм** выбора стратегии существует на DEV, **ограничен N** и снят внешним holdout.
- **Точная формулировка (R2-A3/A6, без overclaim):** сам **selection-holdout сохраняет** best-of-N смещение
  (~0.002–0.003 AUC, SPIKE Эксп. A) — оно **внутреннее** для DEV. Внешний holdout (ADR-0029) архитектура
  **изолирует** (оставляет нетронутым → несмещён), а **не «корректирует»** им внутреннее смещение. То есть честный
  финальный скор — внешний holdout; арбитражный `arb_score` победителя на selection-holdout **слегка оптимистичен**
  (на величину inner-смещения) и таковым подаётся в отчёт.
- **Малая selection-выборка** (spike B) → **runtime-WARNING + guard** (фикс R1-A7): порог по **абсолютному**
  размеру `|sel_holdout|` (валидатор конфига его не знает — зависит от данных), проверяется в арбитре; при
  `|sel_holdout| < 300` (из spike B: ниже — смещение ~0.005) — WARNING «арбитраж зашумлён; уменьшите
  `selection_holdout`/число стратегий или сравнивайте на большем DEV». Режим валиден (внешний holdout несмещён).
  nested-CV арбитража (полное снятие inner-оптимизма) — **будущая** опция (M6d), не дефолт M6c.

### 3. Реакция на timeseries-дрейф (R-FSC-TS-DRIFT)
Для `scheme=timeseries` `sel_holdout` — **поздний** срез `dev_fit` (future-window), **наследующий `purge`
из `CVConfig`** (тот же `purge`-gap, что у внешнего holdout ADR-0029) → нет утечки прошлого в будущее окно.
**`embargo` к одиночному trailing-carve неприменим** (это понятие CV-фолда — gap *после* test перед последующим
train; за holdout обучения нет), поэтому, как и `outer_holdout_carve`, carve использует **только `purge`** —
паритет с внешним holdout (уточнено по ревью кода). Арбитраж предпочитает subset, робастный к **сдвигу во времени**.
`score_subset` для `sequential` строит OOF-фолды **один раз** на `dev_fit` по той же scheme — **без** двойного
carve. Ограничение: фикс-окно `sel_holdout` может недооценивать OOD-дрейф; expanding-window арбитраж — future (M6d).

### 4. Порядок в `run_slice` (фикс R2-MUST-FIX-2/3, FE/калибровка/band/ошибки)
Compare встроен в `run_slice` **до** цикла кандидатов — как одиночный FS в M6b — поэтому всё ниже по потоку
видит **уже спроецированный** на subset-победителя `x_full`:
1. **FE (M6a) — один раз до compare:** инженерия признаков выполняется **однократно** на `dev_fit`; **все** N
   стратегий видят **одну и ту же** FE-матрицу; subset — редукция признакового пространства, **не** переобучение
   FE per-strategy (R2-completeness). Цепочки «FE поверх отбора» нет (как M6b).
2. **compare → калибровка (R2-MUST-FIX-2):** FS-compare выбирает subset-победитель **до** калибровки (ADR-0030/
   0037). Калибратор `best_model_id` считается на **спроецированном на subset** OOF-пространстве (post-freeze) →
   калибратор и refit видят **одни и те же** колонки; рассинхрона входа калибратора нет. (Тождественно M6b, где
   одиночный FS тоже предшествует калибровке; здесь — явно зафиксировано.)
3. **band/значимость — после FS:** значимостная полоса (ADR-0041) применяется к лидерборду, посчитанному на
   **subset**-признаках победителя (FS — пре-процессинг до цикла кандидатов), не к полному пространству.
4. **Ошибка стратегии (R2-MUST-FIX-3) — fail-fast:** если **любая** стратегия в `compare` бросает при отборе
   (`null_importance`/`sequential` на патологии) → `compare_features` **не глушит**: поднимает
   `FeatureSelectionError` с **именем** упавшей стратегии (агрегирует исходное исключение). **Нет** тихого
   выбрасывания стратегии (политика проекта «no silent defaults»). Тесты: одна-падает / все-падают.
5. **Иммутабельность:** `compare_features`/стратегии **не мутируют** вход (`x`/`y`/`folds`/`config`); per-strategy
   `RandomState` независимы (ADR-0049 §1) → compare безопасен для будущей параллелизации без блокировок.

## Последствия
- (+) Внешний holdout остаётся честным якорем (spike-доказано); честное сравнение **разнородных** стратегий
  (ranker vs wrapper) на независимой поверхности; переиспользование `outer_holdout_carve`/`_score_dataset`;
  N=1 ⇒ нет carve, тождественно M6b.
- (−/компромисс) Compare-режим тратит часть DEV на `sel_holdout` (временно, возвращается на refit); шум арбитража
  на малых выборках (WARNING); +N дешёвых фитов рэнкер-модели на арбитраж.
- **Влияние на слои:** арбитр — чистая функция в `application` (тест на фейке); carve — переиспользуемый адаптер;
  внешний holdout-поток (facade) **не меняется**.

## Проверки
- compare-режим вырезает `sel_holdout` из DEV (scheme-aware); каждая стратегия отбирает только на `dev_fit`
  (`sel_holdout`-строки не входят в её внутренний отбор) — тест.
- арбитр детерминирован (фикс. seed → тот же победитель); поверхность независима (тот же subset побеждает вне
  зависимости от того, какой стратегией порождён) — тест на фейк-стратегиях.
- внешний holdout-скор **не** зависит от числа сравниваемых стратегий (несмещённость не регрессирует) — тест.
- малый `sel_holdout` → WARNING; N=1 → carve не выполняется (тождественно M6b).
