# ADR-0044 — Анти-ликедж отбора: OOF-рэнкинг на единых фолдах + агрегирование + дефолт-каталог стратегий

- **Статус:** Accepted (реализован в M6b, 2026-06-10; рэнкер-модель — sklearn `ExtraTrees`, impl-note §2)
- **Дата:** 2026-06-09
- **Драйверы:** DM-1 (анти-ликедж — headline), DM-6 (стоимость); FR-FS-3/5, NFR-FS-1/3/5. Прямое зеркало
  cross-fit-прецедента `crossfit_encode`/`crossfit_calibrate` (ADR-0041/0030); единый набор фолдов `run_slice`
  (ADR-0010); важности моделей через role-interface `SupportsFeatureImportance` (ADR M3).
- **Воркстрим:** M6b (Feature Selection).

## Контекст
Отбор по таргету — ликедж-опасен: рэнкинг на full-train заносит таргет всех строк (вкл. будущие test-фолда) в
решение «какие признаки оставить» → оптимистичный OOF и нечестный отбор (R-FS-LEAK). Наивные реализации именно так
и делают (рэнкинг на полном train, собственный CV ≠ оценочные фолды, R-FS-FOLD-ALIGN). Нужен честный OOF-рэнкинг,
**согласованный** с оценочными фолдами, и дешёвый дефолт-каталог без внешних зависимостей.

## Рассмотренные варианты
1. **Глобальный рэнкинг на full-train, затем оценка тех же фолдов.** Протекает (R-FS-LEAK). **Отвергнут.**
2. **Рэнкинг по собственному TSCV рэнкера.** Фолды ≠ оценочные → отбор «подсматривает» eval-test
   (R-FS-FOLD-ALIGN). **Отвергнут.**
3. **OOF-рэнкинг на ЕДИНЫХ фолдах `run_slice`** (фит на `fit ⊕ es`, агрегирование по фолдам) → один subset.
   Зеркало `crossfit_encode`. **Выбран.**
4. **Вложенный CV отбора внутри каждого train-фолда** (полное устранение остаточного эффекта). Дорого/сложно;
   прецедент (TE/калибровка) использует одиночный cross-fit. **Отложен** (R-FS-RESIDUAL, документировано ниже).

## Решение

### 1. OOF-рэнкинг на единых фолдах (оценка)
Для каждого **оценочного** фолда `f` (тот же `folds = splitter.split(dataset)`, что использует `run_slice`):
```
train_idx_f = fit_idx_f ⊕ es_idx_f            # БЕЗ test_idx_f — те же строки, что у кандидатов
imp_f = ranker.rank(x_full[train_idx_f], y[train_idx_f], categorical=…, random_state=…)
```
Спайн (`select_features`, ADR-0043 §2) **агрегирует среднее по-фолдно нормированных** важностей:
`agg = mean_f( normalize(imp_f) )`; затем отсечение (§3) → **один** subset, общий всем кандидатам и refit.
**Масштаб-инвариантность (фикс ревью A4):** сырые важности GBDT (gain/split count) **не сопоставимы** между
фолдами по магнитуде (разный размер/дисперсия `train_idx_f`) → наивная сумма сместила бы subset к фолдам с
большей магнитудой. Поэтому каждый неотрицательный `imp_f` приводится к доле (`imp_f / imp_f.sum()`) **до**
суммирования; знаковые скоры `random_probe` (margin) уже фолд-относительны (сравнение с зондами того же фита) и
проходят без перенормировки. **Инвариант фолдов (R-FS-FOLD-ALIGN):** отбор использует **тот же** `folds`/splitter,
что оценка — согласование по построению (отдельный `oof_fold_index` не нужен: отбор оперирует
`fold.fit_idx`/`es_idx` напрямую, в отличие от per-row TE). Спайн — чистый numpy в `application` (Humble Object,
NFR-FS-2), тестируется на фейк-рэнкере без обучения.

### 2. Дефолт-каталог (две самодостаточные стратегии, без внешних зависимостей)
- **`importance`** (дефолт): рэнкер-адаптер фитит **дешёвую дефолт GBDT-модель** (LightGBM) на поданной
  обучающей матрице, важности — нативный `feature_importances_` этой rank-модели (per-feature; sklearn-атрибут
  rank-модели, `feature_rankers.py`). Порт `SupportsFeatureImportance.feature_importances` — контрактная
  поверхность для zoo/plugin-эстиматоров, не путь FS. cutoff по умолчанию `top_frac=0.5`. Не требует внешних
  пакетов (LightGBM уже в зависимостях).
- **`random_probe`**: к матрице добавляются `n_probes` случайных колонок (seeded из `random_state`); фит той же
  GBDT; для каждого **реального** признака `margin = importance − max(importance по зондам)`. Спайн усредняет
  margin по фолдам; cutoff `auto` оставляет признаки со **средним margin > 0** (бьёт случайный шум). Дёшево,
  без внешних зависимостей. Детерминирован при фиксированном `random_state` (NFR-FS-3).

Рэнкер-модель — **отдельная** дешёвая дефолт-модель, не кандидаты (ADR-0043 §4). importance/probe —
**task-kind-agnostic** (важности есть и для регрессии). **Отложено в M6c (аддитивно, тот же порт):** `shap`
(внешняя extra), `null_importance` (≥80 перестановок таргета — дорого), `sequential` (обёрточная BE — дорого).

> **Impl-note (реализация M6b):** конкретная рэнкер-модель — sklearn **`ExtraTrees`** (`n_estimators=100`),
> а не LightGBM (имя выше иллюстративно). Причина: sklearn — hard-dependency, поэтому дефолт-каталог
> **extra-free** (NFR-FS-5: `import honestml` не тянет boosting-extra/`shap`) и легче 300-tree boosting; коды
> категорий подаются как numeric (как и кандидаты M3). Архитектурное решение (отдельная дешёвая
> estimator-agnostic рэнкер-модель) не меняется.

### 3. Политика отсечения (cutoff) + floor ≥1
`apply_cutoff(agg, cutoff, min_features)` (чистый, в `application`; `agg` — средний **нормированный** скор, §1):
- `top_k` — `k` старших по `agg` (`k = min(top_k, n_features)`);
- `top_frac` — старшая доля `ceil(top_frac · n_features)`;
- `auto` — `agg > threshold`: для `random_probe` порог `0` (средний margin > 0 ⇒ бьёт шум); для `importance`
  (L1-нормирован, всегда `> 0`) порог — **равномерный базис** `1/n_features` (важность выше «средней доли»).
**Floor (FR-FS-5, R-FS-EMPTY):** результат **никогда** не пуст — если отсечение даёт 0, оставляем `max(1,
min_features)` старших по `agg` (инвариант `design_matrix` §F9 ≥1 признак сохранён). Усечение/floor логируют
**WARNING** (в `apply_cutoff`, единое место) с числом оставленных/отброшенных. Subset сохраняет
**относительный порядок** `schema.features` (индексы отсортированы по позиции, не по рангу — FR-FS-7, NFR-FS-3).

### 4. Где какой отбор применяется (оценка vs refit/inference)
**Чёткое разделение путей** (зеркало ADR-0041 §3):
- **Оценка (цикл кандидатов `run_slice`):** subset вычислен на едином `oof`-рэнкинге; `x_full` **проецируется**
  на subset-колонки **до** цикла → leaderboard/band считаются на отобранных признаках (FR-FS-3).
- **refit / inference:** **тот же** subset (детерминированный исход прогона) сериализуется в схему (ADR-0045);
  `refit_best` учится на subset, `predict` проецирует subset → train(refit)==inference. Рэнкинг на inference
  **не** пересчитывается.

### 5. Честная граница (R-FS-RESIDUAL) — без overclaim, остаток ≥ TE
Одиночный OOF-рэнкинг устраняет **прямую** утечку: ранги фолда `f` построены без его `test`-строк → перестановка
таргета в `test_f` их не меняет (property §Проверки). **Остаётся** остаточный эффект: **единый** subset выбран
по таргету через **все** фолды и затем используется на оценке тех же фолдов → косвенный межфолдовый обмен через
сам факт выбора признаков.

**Честно: эффект потенциально СИЛЬНЕЕ, чем у TE (фикс ревью A2/A8), — не равен/не слабее.** Две причины:
(1) **глобальность** — TE меняет один входной признак локально (per-category карта), а отбор — это **глобальная
маска по всем признакам**, применяемая ко **всем** кандидатам на тех же фолдах; (2) **шумность одиночного фита**
— subset выводится из `folds × 1` рэнкер-фитов; важности одной модели на фолд зашумлены → subset может быть
нестабилен между seed'ами, что усиливает межфолдовый обмен. Аналогия с калибровкой **неполна** (отбор меняет
**вход**, не выход). Сильнее тем: меньше фолдов / выше доля отсекаемых признаков / выше дисперсия важностей.

**Диспозиция — принятый известный компромисс:** одиночный cross-fit — практический стандарт (как TE/калибровка);
полное устранение — **вложенный CV отбора** (nested-CV внутри каждого train-фолда), дорого, отложено (M6c,
future ADR). **Митигации (обязательны):** (а) **holdout (ADR-0029)** — независимая несмещённая проверка поверх:
holdout вырезается **до** `run_slice`, subset выбирается **только** на DEV (OOF) и **замораживается**; отгружаемая
модель refit'ится на full-DEV с этим subset, holdout кодируется/скорится **этим** subset, но **не участвует** в
его выборе → holdout-скор не смещён отбором; (б) **диагностика стабильности** — overlap subset между seed'ами
(Jaccard) как индикатор шумности. **Не выдаётся за zero-leakage** (как ADR-0041 §5) и явно слабее по гарантии,
чем прямой анти-ликедж.

## Последствия
- (+) Честный OOF-отбор, согласованный с оценкой (единый `folds`); дешёвый каталог без внешних зависимостей;
  floor гарантирует ≥1 признак; чистый numpy-спайн (тест на фейке); task-kind-agnostic.
- (−/компромисс) Остаточный эффект единого subset (принят, документирован; holdout поверх); рэнкер-модель ≠
  кандидаты; дорогие стратегии отложены.
- **Влияние на слои:** спайн+cutoff — `application`; стратегии (фит GBDT, важности/probe) — `adapters`; порт —
  `core`.

## Проверки
- **Property (анти-ликедж, NFR-FS-1):** перестановка таргета **в test-части** фолда `f` **не меняет** `imp_f`
  (рэнкинг `f` — из `fit ⊕ es`); зависимость рангов фолда от его собственного test-таргета отсутствует.
- Фолды отбора тождественны оценочным (тот же `folds`) — тест.
- `importance`/`random_probe` на фейк-данных = ручной расчёт (golden); `random_probe` с фикс. seed
  детерминирован.
- `apply_cutoff`: каждая политика даёт ожидаемый размер; обнуляющий случай → ≥1 (floor) + WARNING.
- **Масштаб-инвариантность агрегации (фикс ревью A4):** домножение важностей одного фолда на константу `c>0`
  **не меняет** subset (нормировка по фолду) — golden-тест на фейк-рэнкере с фолдами разной магнитуды.
- **Стабильность subset (диагностика A8):** overlap subset между несколькими seed'ами (Jaccard) фиксируется в
  тесте как индикатор шумности (не строгий инвариант — сигнальный порог).
- Число фитов рэнкер-модели = `folds × 1` (счётчик, NFR-FS-5); **бюджет:** счётчик trials не растёт от FS
  (ADR-0043 §4); holdout-несмещённость (ADR-0029) не регрессирует (subset выбран на DEV-OOF, holdout вне выбора).
