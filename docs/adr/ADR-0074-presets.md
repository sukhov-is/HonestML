# ADR-0074 — Пресеты: декларативные профили поверх фасада

- **Статус:** Accepted (правки по находкам дизайн-ревью R1: поверхность сужена до
  None-дефолтных параметров; значения пресетов — plain-dict JSON-примитивы)
- **Драйверы:** DM-D1, DM-D2 (02-drivers.md); P14, F3.3

## Контекст

Фасад несёт 15 параметров (verbatim, sklearn clone-инвариант; валидация/резолв в
`fit`). Нужны именованные профили «одной строкой» без ветвлений в коде (P14) и без
слома clone/get_params. Эталон приоритета — AutoGluon («user-specified arguments
override presets»), но sklearn-инвариант не видит ФАКТ передачи аргумента — только
значение. Поэтому правило «заполни параметры на дефолтах» безопасно лишь там, где
дефолт = `None` («не задано» и дефолт совпадают семантически); для параметров с
содержательным дефолтом (`significance="bootstrap"`, `finalize=True`,
`run_mode="full"`) оно молча перекрывало бы явный выбор пользователя, равный дефолту
(находка R1).

## Варианты

1. **Поверхность пресета = только None-дефолтные параметры; пресет заполняет
   параметры, оставленные `None`; значения пресета — plain-dict JSON-примитивы.** ✅
2. Та же идея + параметры с содержательными дефолтами — дыра молчаливого
   переопределения (R1-major). Отклонено.
3. Сентинел-дефолты `_UNSET` на всём фасаде — различает «не передал», но меняет
   публичные дефолты и шумит в get_params. Отклонено (цена > ценности v1).
4. Пресеты как классы / применение в `__init__` — ломают единый фасад / verbatim.
   Отклонено.

## Решение

### §1. Форма

`AutoML(preset: str | Mapping[str, Any] | None = None)` — verbatim (Mapping хранится
по ссылке, как list-параметры sklearn; читается снапшотом в начале `fit`). Строка —
ключ встроенного реестра `PRESETS` (`composition/presets.py`); Mapping —
пользовательский пресет той же формы (YAML пользователь грузит сам — зависимость не
вводится). Неизвестное имя → `ConfigError` со списком известных; ключ вне поверхности
→ `ConfigError` (опечатки и попытка пресетить `significance` не молчат).

**Поверхность пресета — ровно None-дефолтные параметры фасада (7):** `cv`, `models`,
`budget`, `hpo`, `ensemble`, `feature_selection`, `feature_engineering`.
Вне поверхности: `task`/`metric` (доменный выбор), `random_state`
(воспроизводимость), `significance`/`finalize`/`run_mode` (содержательные дефолты —
зона дыры R1), `cache`/`tracker` (инфраструктурные opt-in).

**Значения — plain-dict/JSON-примитивы** (симметрия built-in/Mapping/YAML; P14
«данные, не код»): `{"cv": 3}`, `{"hpo": {}}`, `{"ensemble": {"n_bags": 10}}`.
Нормализация в `_resolve_preset`: для конфиг-валентных ключей dict →
`Config.model_validate(dict)` (`{}` → дефолтный конфиг; `ValidationError` →
`ConfigError`); `cv`/`budget` принимают и скалярные формы фасада.

### §2. Правило приоритета и применение

В начале `fit`: `_resolve_preset()` строит `eff` — словарь эффективных значений
7 параметров поверхности: `eff[p] = preset[p], если self.<p> is None, иначе
self.<p>`. **None = «не задано, делегирую пресету/дефолту»** — это точная семантика,
а не эвристика (дефолты всех 7 — `None`). Явный opt-out при включающем пресете
(«best, но без HPO») выражается кастомным Mapping (копия профиля без ключа) — этот
эскейп-хэтч документируется в пресет-доке. `self.*` не мутируются (clone-инвариант);
дальнейший `fit` читает 7 параметров только из `eff`: 4 значения уходят аргументами
в существующие резолверы (`_resolve_fe/_resolve_fs/_resolve_hpo/_resolve_ensemble` —
параметризуются значением вместо `self.*`), `cv`/`models`/`budget` — в свои точки
чтения (`_normalize_cv`/`build_default_components`/`_resolve_budget`). Итого
механизм: 4 резолвера + 3 точки чтения; валидация `run_mode` и прочие параметры не
затрагиваются.

### §3. Наблюдаемость (DM-D2)

INFO-лог «preset %s applied: %s»; run_report — аддитивный top-level ключ
`"preset": {"name": str | None, "applied": [имена]} | None` (для Mapping-пресета
`name=None`; без пресета — ключ `None`; `RUN_MANIFEST_VERSION` не бампится).
Механизм передачи в ассемблер: новый **опциональный kwarg `preset=None`** у
`build_run_report` (application получает данные значениями — направление
зависимостей не меняется; legacy-вызовы не трогаются). Fingerprint пресет не несёт —
несёт РЕЗОЛВНУТЫЕ параметры (NFR-DLV-2): пресет — сахар ввода, не семантика рана.

### §4. Встроенные профили v1 (данные; правка = правка dict + CHANGELOG)

| Имя | Содержимое (plain-dict) | Смысл |
|---|---|---|
| `fast` | `{"cv": 3}` | быстрый прототип: меньше фолдов; честность (band) НЕ отключается |
| `balanced` | `{"ensemble": {}}` | дефолты + бленд под significance-гейтом |
| `best` | `{"hpo": {}, "ensemble": {}}` | максимум качества; HPO требует extra `optuna` — честный `MissingDependencyError` (R-DLV-1) |

`models` не фиксируются (`None` → реестр гейтит по доступности extras). Параметры
честности (`significance`/`finalize`) пресетам недоступны по построению §1 —
пресет не может молча понизить честность (R1-находка про `fast`).
**Zero-shot/learned-портфели — отложены** (P15): требуют офлайн-инфраструктуры
уровня TabRepo; статический рукописный портфель — это просто ещё одна запись данных,
добавляемая post-M9 при спросе.

## Последствия

- (+) Дыра молчаливого переопределения исключена конструкцией (а не оговоркой);
  OCP: новый профиль = запись в dict; формы built-in/Mapping/YAML симметричны.
- (+) sklearn-инвариант цел; воспроизводимость — резолвнутым конфигом.
- (−) Пресеты не могут управлять `significance`/`finalize`/`run_mode` — осознанная
  цена честности; при реальном спросе путь — сентинел-дефолты отдельным ADR.
- (−) Имена и содержимое профилей — публичный контракт (SemVer, CHANGELOG).
