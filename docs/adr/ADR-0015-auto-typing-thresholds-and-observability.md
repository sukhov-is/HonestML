# ADR-0015 — Авто-типизация: пороги как поля `Task` + наблюдаемость решений

- **Статус:** Proposed
- **Дата:** 2026-06-06
- **Драйверы:** D1 (расширяемость), D6 (инженерная закалка); FR-1, FR-2; NFR-2/3.
- **Воркстрим:** F1 (контракт входных данных), дельта к M1a.

## Контекст

Авто-типизация в `Reader` тихо меняет набор фич: id-like числовые → `ColumnRole.IGNORE`,
малокардинальные → `CATEGORICAL`. Два порога уже в `Task` (`numeric_cat_max_unique=20`,
`numeric_id_rate=0.95`), но третий — **`nuniq>100` захардкожен** (`reader.py:174`), а
сами решения **нигде не сообщаются** (Reader без логов). На разных датасетах поведение
меняется непрозрачно для пользователя (F1.7). Ограничение: реклассификация роли
определяет, в какой блок границы попадёт колонка (numpy-числа vs коды категорий,
ADR-0005) — инвариант менять нельзя.

## Рассмотренные варианты

1. **Оставить как есть** — (−) хардкод + молчание, футган доверия (F1.7 не закрыт).
2. **Вынести порог + только INFO-лог** — (+) настраиваемо и видно в логах; (−) нет
   программного доступа к решениям (нельзя проверить тестом/показать в отчёте рана).
3. **Вынести порог + лог + структурированный typing-report** — (+) и лог, и
   программно доступный диагностический объект; (−) +маленькая поверхность (форма отчёта).

## Решение

**Вариант 3.** Третий порог → поле `Task` `numeric_id_min_unique: int = 100` (ge=1),
рядом с существующими; **дефолт = текущее поведение** (NFR-3).

**Семантика порога (R-2 ревью):** `numeric_id_min_unique` — нижняя граница в
**составном** условии id-like, а не самостоятельный классификатор:
`is_id_like = (rate > numeric_id_rate) AND (nuniq > numeric_id_min_unique)`
(как сейчас `reader.py:174`). Реализатор НЕ должен делать его независимым правилом.

**typing-report — формальный контракт (R-1/R-2 ревью):**
- **Наивный baseline** = роль строго по dtype без порогов: целочисленная numeric → `NUMERIC`,
  float → `NUMERIC`, `Utf8`/`Categorical` → `CATEGORICAL`, `Date`/`Datetime` → `DATETIME`.
- В отчёт включаются **только отклонения числового блока** от baseline:
  `numeric → IGNORE` и `numeric → CATEGORICAL`. Нативные `Utf8/Categorical → CATEGORICAL`
  и `Date → DATETIME` **не репортятся** (совпадают с baseline).
- **Закрытый набор `reason`:** `numeric_id_like` (составное id-условие выше) и
  `low_cardinality_int` (`nuniq <= numeric_cat_max_unique`).
- Запись: `(column, source_dtype, assigned_role, reason)`.

**Жизненный цикл отчёта (R-2 ревью):** отчёт доступен **через атрибут `Reader`**
(`reader.typing_report`), **не меняя возвращаемый тип `Reader.read`** (контракт
M2-фасада, RF3). Атрибут **сбрасывается в начале каждого `read`** и отражает **последний**
вызов; на **inference-ветке (передана `schema`)** авто-типизация не выполняется →
отчёт **пуст**. `Reader` **не thread-safe** — один инстанс на поток (документируется).

`Reader` пишет записи отчёта в `logging.getLogger("honestml.adapters.reader")` (потомок
корня `honestml` с `NullHandler`, key=value по ADR-0003) на INFO — без `print`/global
state (NFR-2). Новый порог конфигурируется **через объект `Task`** (не через строковый
shortcut фасада). Инвариант границы numpy+коды (ADR-0005) не затрагивается.

## Последствия

- **Положительные:** прозрачность (видно, какие колонки и почему реклассифицированы) и
  полная настраиваемость порогов; дефолты не меняют поведение M1a.
- **Отрицательные / компромиссы:** новое поле в публичном `Task` — contract change, но
  **аддитивный**, с round-trip-совместимостью (RF2, NFR-4); форма typing-report
  становится мини-контрактом — зафиксирована минимальной (RF3).
- **Влияние на слои/границы:** правки только в `core/task.py` (поле + валидация) и
  `adapters/reader.py` (применение порога + сбор/лог отчёта); `FeatureSchema` и граница
  данных без изменений.

## Проверки

Тест: смена `numeric_id_min_unique` меняет реклассификацию на синтетике, дефолт держит
текущее поведение; тест содержимого `typing_report` (id-like → IGNORE, cat-like →
CATEGORICAL с верной reason) и наличия INFO-лога; эквивалентность `to_numpy()`/
`categorical_codes()` при дефолтах; round-trip `Task` со снимком без поля
(`tests/unit/test_reader_dataset.py`, `test_task.py`). ruff `T201` = 0; import-linter.
