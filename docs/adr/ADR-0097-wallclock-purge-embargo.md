# ADR-0097: Wall-clock (Δt) purge/embargo для time-series схем

- **Статус:** Accepted (реализован, этап 2 — 2026-06-23)
- **Дата:** 2026-06-23
- **Драйверы:** D-4 (duration-корректный зазор для нерегулярных рядов); FR-4, NFR-2.
  Расширяет ADR-0027 (purge/embargo в строках).

## Контекст
Сейчас `purge`/`embargo` — целочисленные **счётчики строк** (`config.py:47-48`,
`splitters.py:409-414`): зазор удаляет N ближайших по порядку строк. На **нерегулярных** рядах
(биржа: ночь/выходные/праздники) N строк соответствуют разному реальному времени — зазор то
недо-, то переочищает. ADR-0027 явно отметил это как компромисс («для нерегулярных — приближение»).
Полный purge горизонта метки по `label_time` уже корректен по длительности (сравнение значений
`t1`), но зазор серийной корреляции (purge перед тестом / embargo после) — нет.

## Рассмотренные варианты
1. **Только `label_time`.** Закрывает горизонт метки, но не зазор серийной корреляции; требует
   знать `t1`. Недостаточно.
2. **Конвертировать число строк в Δt эвристикой** (median Δt × N). Неявно, неточно на
   нерегулярных рядах — ровно то, что чиним. Отвергнут.
3. **Явные duration-знобы `purge_delta`/`embargo_delta`, отсечка по значению времени.** Точно для
   любой регулярности; переиспользует уже существующие value-based фильтры сплиттера. **Выбран.**

## Решение

### 1. `CVConfig` (core, аддитивно)
- `purge_delta: float | None = None`, `embargo_delta: float | None = None` — ширина зазора **в
  единицах оси времени** (для datetime — в тех же единицах, что хранит `dataset.time()`; число —
  как есть). `gt=0`.
- **Единица целочисленного зазора (F5):** `purge`/`embargo` (int) — это зазор в натуральной
  единице схемы: **строки** под `timeseries`, **периоды** под `timeseries_period`. `*_delta` — Δt
  под обеими. Это «другая ось» в смысле взаимоисключимости ниже.
- **Валидатор `CVConfig`:** взаимоисключимость — нельзя задать одновременно целочисленный `purge>0`
  и `purge_delta` (то же для embargo). Field-coherence (не зависит от резолва схемы) → в
  `model_validator`, бросает `ValueError` → у прямого `CVConfig(...)` это pydantic `ValidationError`
  (как Budget/FS), на `RunConfig.parse`/preset-dict — `ConfigError` (G2/F15/R-7).
- **Гейт схемы в `build`** (паттерн `build.py:677-680`): `*_delta` при схеме не из
  `{timeseries, timeseries_period}` → `ConfigError`.

### 2. Сплиттеры (adapters)
Применимо к `TimeSeriesSplitter` и `PeriodTimeSeriesSplitter`. Зазор вычисляется **по значению**
на отсортированной по времени оси:
- **purge_delta:** из train исключаются строки с `test_min_t - purge_delta ≤ t < test_min_t`
  (полуинтервал: нижняя включена, верхняя исключена; обобщение `splitters.py:415-416`). Целочисленный
  `purge` (строки/периоды) и `purge_delta` — взаимоисключимы (валидатор), активна ровно одна семантика.
- **embargo_delta:** из train **поздних** фолдов исключаются строки в зоне
  `[prev_test_max_t, prev_test_max_t + embargo_delta)` после ранних тест-окон (de-Prado серийная
  корреляция; value-based аналог `splitters.py:410-413`). **Полуинтервал согласован с purge**
  (верхняя граница исключающая, F11) — чтобы не возникало несимметричного off-by-one на стыке.
- Инвариант `validate_fold` (value-based overlap) не меняется — он подтверждает отсутствие
  пересечения, а величину Δt-зазора проверяет отдельный splitter-тест (разведение гарантий
  overlap vs magnitude — паттерн ADR-0027 «Проверки»).

## Последствия
- **Положительные:** честный зазор на нерегулярных/биржевых рядах; композируется с ADR-0096 и с
  текущей `timeseries`; нулевой новый механизм проверки (переиспользует value-based фильтр).
- **Отрицательные/компромиссы:** две семантики одного концепта (строки/периоды vs Δt) — решается
  взаимоисключимостью и явными дефолтами (оба None/0 → текущее поведение); для datetime-оси
  единица Δt — это единица хранения `time()` (документируется). Carve-пути одиночного холдаута
  (`outer_holdout_carve`/`_timeseries_carve`) принимают только `purge`/`purge_delta`, но **не**
  `embargo` (embargo бессмыслен для единичного holdout — by design, как и для строкового embargo
  сейчас); применимость `*_delta` относится к per-fold сплиттерам.
- **Влияние на слои:** `core` (поля+валидатор, аддитивно), `adapters` (логика среза),
  `composition` (гейт схемы). Слои/контракты не нарушены.

## Проверки
- На нерегулярной оси `purge_delta=Δ` удаляет ровно строки `[test_min_t-Δ, test_min_t)`
  (magnitude-тест); `embargo_delta` — зону после ранних тестов (FR-4, NFR-2).
- Одновременно `purge` и `purge_delta` → `ValueError`; `*_delta` при не-timeseries → `ConfigError` (FR-9).
- Дефолт (оба не заданы) воспроизводит текущее поведение `TimeSeriesSplitter` байт-в-байт (NFR-5).
