# ADR-0001 — Раскладка пакета и optional extras

- **Статус:** Proposed
- **Драйверы:** D6 (инженерная закалка), R7 (тяжёлые зависимости); NFR-8.
- **Воркстрим:** M0.

## Контекст

Сейчас: нет `pyproject.toml`, только `requirements.txt`, который тянет **все**
тяжёлые зависимости как обязательные (catboost/lightgbm/xgboost/shap/mlflow).
Пакет — плоский `honestml/` (flat-layout). Для OSS это блокер: тяжёлая установка,
невозможность поставить лёгкое ядро, нет границы «ядро vs адаптеры».

## Рассмотренные варианты

1. **flat-layout, все зависимости обязательны** (как сейчас) — просто, но тяжело,
   мешает изоляции ядра, провоцирует случайные импорты тестов из корня.
2. **src-layout + один пакет + optional extras** — стандарт современных
   Python-библиотек; тестируется установленный пакет, ядро лёгкое.
3. **Мульти-пакетный монорепо** (отдельные дистрибутивы `core`/`adapters`) —
   максимально чисто, но преждевременно усложняет релиз/версионирование на старте.

## Решение

Вариант **2**: `src/honestml/` (src-layout), один дистрибутив, `pyproject.toml`
(PEP 621). Import-пакет — `honestml`; имя дистрибутива на PyPI — TBC на M9.

**Ядро (обязательное):** `numpy`, `pandas`, `polars`, `scikit-learn`,
`pydantic>=2`. Только это нужно, чтобы импортировать `honestml` и доменные порты.

**Optional extras** (тяжёлое — за флагами, с capability-detection):

| Extra | Тянет | Назначение |
|---|---|---|
| `catboost`/`lightgbm`/`xgboost` | соответствующий бустинг | модель-адаптеры |
| `boosting` | все три | удобный мета-extra |
| `shap` | shap | SHAP-рэнкер отбора |
| `optuna` | optuna | тюнер-адаптер |
| `mlflow` | mlflow | трекинг-адаптер |
| `onnx` | onnx/onnxmltools/skl2onnx | экспорт |
| `report` | matplotlib и пр. | отчёты |
| `all` | всё пользовательское | «полная» установка |
| `dev` | ruff/mypy/pytest/pre-commit/mkdocs | разработка |

Адаптер при отсутствии своей зависимости поднимает понятную ошибку
(`MissingDependencyError` с подсказкой `pip install honestml[xgboost]`), а не
`ImportError` в импортах ядра. Реестр (ADR воркстрима M3) пропускает
незарегистрированные из-за отсутствующих extras компоненты.

## Последствия

- (+) Лёгкое ядро; контрибьюторы и CI ставят минимум; чистая граница ядро/адаптеры.
- (+) src-layout исключает «случайный» импорт из рабочей директории (текущий
  `conftest.py` хак с `sys.path.insert` уходит).
- (−) Нужна дисциплина: ни один модуль ядра не импортирует extra-зависимость на
  уровне модуля (только внутри адаптеров, лениво) — проверяется import-linter (M1).
- Существующий код `honestml/` мигрирует в `src/honestml/`; это breaking (R5) →
  отражается в мажоре.
