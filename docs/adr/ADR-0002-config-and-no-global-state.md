# ADR-0002 — Типизированная конфигурация и отказ от глобального состояния

- **Статус:** Proposed
- **Драйверы:** D6, D4 (воспроизводимость без global state); NFR-6, NFR-4, A1, A2.
- **Воркстрим:** M0 (вводит механизм; доменные модели наполняются в M1).

## Контекст

Сейчас два антипаттерна:
- **A1:** `config.py` — module-level константы, захардкоженные под один проект
  (доменно-специфичные имена колонок-идентификаторов). Нет валидации, нельзя параметризовать без
  правки исходника, «магические строки».
- **A2:** мутабельное module-level состояние в `selection.py` (`_timings`,
  `_feature_log`) — накапливается между вызовами; не реентерабельно, ломает
  параллельные прогоны и тестируемость.

Индустрия подтверждает риск: PyCaret 4.0 **выпилил** global-singleton именно
из-за неконкурентности.

## Рассмотренные варианты

1. **Оставить module-глобали** — нет.
2. **dataclasses** — типы есть, но нет валидации/коэрции/сериализации из коробки.
3. **Pydantic v2 модели/Settings** — валидация на границе, дефолты, JSON-сериализация
   (нужна для run-manifest, FR-15), вложенность, `Settings` для env. Уже в ядре.
4. **Hydra/OmegaConf** — мощно для CLI-экспериментов, но тяжёлый рантайм и
   YAML-композиция избыточны для библиотеки-API.

## Решение

Вариант **3**. Вся конфигурация — **Pydantic v2** модели:
- Доменно-специфичные имена колонок/роли **уходят из библиотеки** в передаваемый
  пользователем конфиг/`Task`/`FeatureSchema` (детализация — M1). Библиотека не
  знает доменных имён колонок.
- Параметры прогона (CV, бюджет, seed, **порядок тай-брейков/`SelectionPolicy`** —
  не взвешенная сумма по умолчанию, ADR-0007/C-1) — поля типизированных моделей с
  валидацией и дефолтами; передаются **явно** в use-cases.
- **Никакого module-level мутабельного состояния.** Тайминги/логи удаления фич
  (бывшие `_timings`/`_feature_log`) живут в объекте-контексте прогона
  (`RunContext`/`Experiment`), создаваемом на каждый `fit` и передаваемом явно.
- Конфиг сериализуем → основа run-manifest (FR-15) и fingerprint-кэша.

## Последствия

- (+) Реентерабельность и потокобезопасность; несколько `fit` в одном процессе
  независимы (NFR-6).
- (+) Валидация на границе ловит ошибки конфигурации рано; «магические строки»
  заменяются типами/enum.
- (+) Воспроизводимость: весь конфиг — сериализуемый артефакт.
- (−) Сквозной рефакторинг: убрать чтение глобалей по всему коду, протащить
  контекст явно. Делается при миграции модулей в M1–M5.
- Pydantic v2 фиксируется как зависимость ядра (ADR-0001).
