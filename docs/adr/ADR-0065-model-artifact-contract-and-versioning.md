# ADR-0065 — Самоописывающий контракт `ModelArtifact` + политика версионирования

- **Статус:** Accepted (M8, design-gate pending)
- **Драйвер:** DM-82 (контракт-стабильность) — FR-SRV-1; NFR-SRV-2 (← NFR-11, G-A1)
- **Связано:** ADR-0012 (артефакт-каталог + `artifact_version`, baseline), ADR-0024 (classes/manifest-ключи),
  ADR-0005 (FeatureSchema = единый препроцессинг), ADR-0064 (ensemble-блок). Готовит seam для M8b
  (native/ONNX `model_type`).

## Контекст
Артефакт-каталог версионирован (`ARTIFACT_VERSION=1`) и эволюционирует **строго аддитивно** через
`manifest.get(key, default)`; обязательны только `task`/`metric`/`best_model_id` + гейт `artifact_version`
(cross-version → `SchemaValidationError`). Но: (1) **политика совместимости не специфицирована** сверх «равно/
не равно» (G-A1/NFR-11 требует явного in-major/cross-major контракта и forward-load полей); (2) `model_type`
— константа `"joblib"`, **на загрузке игнорируется** → артефакт не диспетчеризуем, не готов к нативным/ONNX
форматам M8b; (3) самоописание частично — нет явного перечня «что манифест обязан нести для standalone-
реконструкции». Нужен формализованный контракт без бампа версии (дельта M8 аддитивна).

## Рассмотренные варианты
1. **Оставить как есть** — ❌ NFR-11/G-A1 не закрыт: cross-major-поведение и forward-load не специфицированы;
   `model_type`-dispatch отсутствует → M8b упрётся в неконтрактный артефакт.
2. **Единый блоб (pickle всего фасада)** — ❌ отвергнут ещё ADR-0012 (FLAML-хрупкость): не инспектируемо,
   усиливает RCE-поверхность, ломает посхемный препроцессинг.
3. **Формализовать каталог как самоописывающий контракт + явная политика версий + load-bearing `model_type`**
   — ✅ манифест декларирует полный набор для реконструкции; in-major аддитивно / cross-major явный отказ;
   `model_type` диспетчеризует загрузку модели (этот проход — только `"joblib"`, native/onnx → M8b).

## Решение (Вариант 3)

### §1 Манифест = самоописывающий контракт
Манифест ОБЯЗАН нести всё для standalone-реконструкции `FittedModel` (без обучающего стека):
`artifact_version`, `task`, `metric`(+`metric_average`), `classes`, `model_type`+`model_file`,
`calibrator_file`, ensemble-блок, препроцессинг — через `schema.json` (`FeatureSchema`). Каталог-форма
(набор файлов, не блоб) сохраняется: `schema.json` человекочитаем, нативная модель — отдельный файл (готовит
M8b). Добавляемые этим проходом ключи (аддитивно): `checksums` (ADR-0067), `shipped_on`/`finalize` (ADR-0068).

### §2 Политика версионирования (явная)
- `ARTIFACT_VERSION` = **major** контракта. **In-major** (тот же major): новые ключи **только аддитивны**,
  читаются через `manifest.get(key, default)` с «легаси»-дефолтом → forward/backward-симметрия (старая сборка
  читает новый артефакт по известным ключам; новая читает старый по дефолтам).
- **Cross-major** (`found_major != reader_major`): **явный отказ** `SchemaValidationError` с found/expected и
  действенным сообщением. Бамп major допустим **только** при не-аддитивном изменении (удаление ключа /
  смена семантики), сопровождается записью в `operational.md` и (при необходимости) миграцией.
- Этот проход **остаётся `ARTIFACT_VERSION=1`** — вся дельта аддитивна. `RUN_MANIFEST_VERSION`/
  `FINGERPRINT_VERSION` независимы и не затрагиваются.
- **Объём (правка R1):** «major» сейчас **тождественен** всему `artifact_version` (единственный major=1) →
  гейт в M8 остаётся **точным сравнением** `found != ARTIFACT_VERSION` (не вводить парсинг major из int).
  Выделение major в отдельное поле / семвер — **Day-2**, в объём прохода не входит (scope).

### §3 `model_type` — load-bearing dispatch
`load_artifact` диспетчеризует загрузку модели по `manifest["model_type"]`: `"joblib"` → `joblib.load`
(единственный реализуемый в M8 фундаменте). Неизвестный тип → **явная ошибка** (а не молчаливый joblib).
Это seam для M8b (`"catboost_native"`/`"onnx"` и т.п.) — добавление нового типа аддитивно, старый ридер
встретит неизвестный тип → честный отказ (cross-capability, не cross-version). **Порядок проверок (R1):**
version-gate (§2) → integrity-verify (ADR-0067 §2) → `model_type`-dispatch — с **различимыми** сообщениями
(несовместимая версия / повреждение-целостности / неизвестный тип модели), чтобы причина отказа была явной
(NFR-SRV-5).

## Последствия
- **+** NFR-11/G-A1 закрыт: совместимость специфицирована и тестируема; cross-major — явный отказ.
- **+** Артефакт самодостаточен и диспетчеризуем → готов к M8b без слома контракта; `ARTIFACT_VERSION` остаётся 1.
- **+** Минимальная дельта: формализация существующего каталога, не новый формат (NFR-SRV-6).
- **−:** `model_type`-dispatch — небольшое изменение `load_artifact` (раньше игнор) → нужен тест на
  неизвестный тип; legacy-артефакты несут `model_type="joblib"` (совместимы).
- **−:** «major» из одного int — простая схема; semver-манифест/`min_reader_version` как поле → Day-2 (если
  понадобится тоньше).

## Day-2 (committed)

- `min_reader_version`/semver-поле манифеста, если потребуется более гранулярная совместимость.
- PII-минимизация полей манифеста/`CategoryTable` (NFR-18/G-P1) → отдельный проход.
