# ADR-0082 — Честный expanding-window OOF target-encoding под time-series CV

- **Статус:** Accepted (реализован 2026-06-16; снимает defensive-skip C1/C4 аудита 2026-06-14)
- **Дата:** 2026-06-15
- **Драйверы:** DM-1 (анти-ликедж — headline). Закрывает **остаток C1/C4** аудита 2026-06-14
  (полноценное лекарство вместо защитного skip). Дельта к ADR-0041 (OOF TE на едином `oof_fold_index`):
  распространяет его на time-ordered scheme. Опирается на ADR-0027 (TimeSeriesSplitter: value-based
  порядок, expanding-окно, гарантия нулевого пересечения по времени), ADR-0040 (FE-пайплайн).
- **Воркстрим:** post-audit (v0.2+).

## Контекст
ADR-0041 строит честный OOF target-encoding кросс-фитом `crossfit_encode`: фолд `f` кодируется картой,
обученной на `oof_fold_index != f` — то есть на **всех ДРУГИХ фолдах, включая более ПОЗДНИЕ**. Под IID это
корректно (ADR-0041 §5: косвенный обмен распределением между фолдами — принятый компромисс). Под
**time-ordered CV** (ADR-0027) «другие фолды» включают БУДУЩЕЕ относительно теста → look-ahead прямо в
признаке (аудит 2026-06-14, C1/C4).

Текущая мера (по результату аудита) — **защитный skip**: facade `_resolve_fe` выключает TE целиком при
`scheme=="timeseries"`, плюс defensive-гейты в `run_slice`/`tune_estimators` (сбрасывают `te_on=False` при
`getattr(splitter,"time_ordered",False)`). Это устраняет утечку, но ценой полной потери TE на time-series
задачах — самого ценного FE. Нужно **честное** кодирование, использующее только прошлое.

## Рассмотренные варианты
1. **Сохранить skip (статус-кво).** Честно, но TE недоступен под time-series. Отвергнут как постоянное
   решение (был временной мерой аудита; см. ADR-0041 §4 — гейт мыслился как graceful, не как финал).
2. **Per-row expanding (running statistics).** Каждая строка кодируется из строго более ранних по ВРЕМЕНИ
   строк (running smoothed mean). Максимально точно, но дорого и НЕ выровнено с фолд-структурой оценки
   (риск рассинхрона `oof_fold_index`, R-FE-FOLD-ALIGN). Отвергнут (over-engineering; на фолд-гранулярности
   честность уже достигается).
3. **Expanding по фолдам на ЕДИНОМ `oof_fold_index`: фолд `f` кодируется из строго более ранних фолдов
   (`idx < f`).** Выбран — честно, дёшево (та же машинерия и единый индекс, ADR-0041 §1), выровнено с
   оценочными фолдами.
4. **Nested-CV TE внутри каждого train-фолда.** Строже устраняет остаточный train-feature эффект, но дорого;
   ADR-0041 §5 уже отложил nested как future. Не вводим здесь.

## Решение

### 1. `crossfit_encode_expanding` — фолд кодируется строго из более ранних
Новая чистая функция в `application/feature_encoding.py` (зеркало `crossfit_encode`, ADR-0041 §1), но
**fit-множество фолда `f` = строки строго более ранних фолдов**. На том же `oof_fold_index`, что использует
`run_slice` (R-FE-FOLD-ALIGN не нарушается):

```
для фолда f >= 0:  карта_f = smoothed_mean( codes[idx < f], y[idx < f] )
                   OOF_TE[idx == f] = карта_f.lookup( codes[idx == f] )   # global_mean_f для непокрытого кода
```

**Почему `idx < f` корректно даёт «всё строго раньше по времени».** `TimeSeriesSplitter` (ADR-0027 §1)
нарезает тест-окна как последние `n_splits·n_test` строк в порядке времени, последовательными окнами; фолд
`k` yield'ится `k`-м, а `_fold_index` нумерует фолды порядком их выдачи → **`fold_id` монотонен по времени**.
Начальный train-блок (строки до первого тест-окна) никогда не является test → в `_fold_index` всегда `-1`, и
он строго раньше всех тест-окон. Так как `-1 < f` для любого `f >= 0`, условие `idx < f` захватывает
`(-1 блок) ∪ (фолды 0..f-1)` — ровно «всё, что строго раньше тест-окна `f`».

**Инвариант (анти-look-ahead, NFR-FE-1 для time-series):** OOF-TE строки фолда `f` зависит только от строк
строго РАНЬШЕ её тест-окна → не видит ни свой таргет, ни любой таргет из фолда `f` и позже. Доказуемо
property-тестом (см. §Проверки).

### 2. Граница: самый ранний блок (`idx == -1`)
У `-1` блока нет строго более ранних данных (`idx < -1` пусто). Эти строки — **train-only** (никогда не в
оценочном OOF; служат только как train-признак в фолдах, где входят в `fit_idx`), поэтому их кодировка **не
влияет на честность leaderboard-скора** (он считается на фолдах ≥0, закодированных из прошлого). Кодируем их
**базовой ставкой блока** — `global_mean = mean(y[idx == -1])` (единственные не-будущие данные), одинаково для
всех колонок и категорий. Это документируемый граничный компромисс (как ADR-0041 §5), а не per-категорийный
сигнал. Реализационно: для минимального fold-значения, если `idx < f` пусто → `out[test] = mean(y[test])`.

Альтернатива — self-encode `-1` блока (per-категорийная карта на самом блоке) — даёт больше train-сигнала, но
это оптимизация train-признака вне рамок честности; **отложена** (можно добавить позже без смены контракта).

### 3. Сглаживание и резервы — без изменений (ADR-0041 §2)
`smoothed = (cnt·mean + k·global_mean_f) / (cnt + k)`, `k = FEConfig.te_smoothing`. Непокрытый/unseen код,
включая `null_code`/`unknown_code` (через `reserve_from`) → `global_mean_f`. Отличие от ADR-0041 — только
fit-множество: `global_mean_f = mean(y[idx < f])` (per-fold, как и раньше, но на прошлом).

### 4. Маршрутизация: снять defensive skip, включить expanding
- **facade `_resolve_fe`:** убрать ветку disable при `scheme=="timeseries"`. Бинарный гейт (ADR-0041 §4:
  `task.kind != "binary"` → skip+WARNING) **остаётся**. Reader снова материализует full-train `*_te` —
  корректно для refit/inference (весь DEV в прошлом относительно inference; асимметрия «OOF для оценки /
  full-train для отгрузки» ADR-0041 §3 сохраняется; holdout-несмещённость ADR-0029 не регрессирует, т.к.
  holdout вырезается до всего, а спека фитится только на DEV).
- **`run_slice`:** вместо `if te_on and time_ordered: te_on = False` — пробросить `time_ordered` в
  `_augment_oof_te`, оставив `te_on = True`.
- **`tune_estimators`:** аналогично по `getattr(inner_splitter, "time_ordered", False)`.
- **`_augment_oof_te`:** добавить флаг `time_ordered: bool` — маршрут на `crossfit_encode_expanding` (True)
  vs `crossfit_encode` (False, IID — без изменений).

Это **заменяет** защитные гейты C1/C4 (skip → expand). Defensive-комментарии и сбросы `te_on`, добавленные при
аудите, удаляются (их роль теперь исполняет честный expanding-путь).

### 5. Честная граница точности (зеркало ADR-0041 §5, для time-series)
Expanding устраняет **look-ahead**: OOF-TE фолда `f` строится только из фолдов `< f`. Остаются честно
задокументированные остаточные эффекты:
- `-1` блок кодируется базовой ставкой (train-only компромисс, §2).
- **purge/embargo НЕ применяются к fit-множеству энкодера.** Фолд-гранулярность `idx < f` и так строго раньше
  тест-окна `f` (нулевое пересечение по времени гарантировано составом фолдов, ADR-0027 §2), поэтому
  look-ahead отсутствует; недо-пёрдж энкодера лишь делает TE **богаче данными**, а не протекающим (серийная
  корреляция у самой границы притупляется сглаживанием `k`). Документируется (применён ≠ под-пёрджен), не
  выдаётся за zero-leakage.
- Ранние фолды видят меньше прошлого (expanding) → их TE шумнее; `k` сжимает к `global_mean_f`. Holdout
  (ADR-0029) — независимая несмещённая проверка поверх.

Объём — **бинарная классификация** (наследует гейт ADR-0041 §4). Multiclass/regression остаются skip+WARNING.

## Последствия
- (+) Честный target-encoding под time-series; убирает потерю FE от defensive skip; переиспользует
  `oof_fold_index` и smoothing-контракт ADR-0041; чистый numpy (тест на фейках).
- (−/компромисс) **Меняется численность** прогонов с `scheme="timeseries"` (TE теперь активен) — затрагивает
  showcase house-prices и любые time-series конфиги; `-1`-блок кодируется base-rate; TE ранних фолдов шумнее.
- **Влияние на слои:** `crossfit_encode_expanding` — `application` (numpy, без polars/sklearn); маршрут флага —
  `application/slice` + `application/tuning`; снятие гейта — `composition/facade`. import-linter не нарушается.
- **Замена прошлых решений:** аннулирует defensive-меры C1/C4 (skip TE под time-series) — они становятся
  expanding-путём; ADR-0041 §4 timeseries-disable снимается (бинарный гейт остаётся).

## Проверки
- **Golden:** `crossfit_encode_expanding` на фейк-массивах = ручной расчёт (фолд `f` из `idx < f`; `-1` блок →
  base-rate; reserve-коды → `global_mean_f`).
- **Property (expanding анти-look-ahead, NFR-FE-1):** перестановка таргета в фолде `f` **И во всех более
  поздних** фолдах НЕ меняет OOF-TE строк фолда `f` (их кодировка — из `idx < f`).
- **facade:** binary + `scheme="timeseries"` + TE → TE **АКТИВЕН** (expanding-путь), не skip (обновляет
  регресс-тест аудита `test_te_timeseries_skips_with_warning`); multiclass/regression + TE → skip+WARNING
  (без изменений).
- **run_slice / tune_estimators:** `time_ordered` splitter → expanding-путь (обновляет
  `test_te_skipped_under_time_ordered_splitter`); IID splitter → обычный `crossfit_encode` (регресс, бит-в-бит).
- Детерминизм при фиксированном seed; число фитов TE = `folds × cat_cols` (счётчик, NFR-FE-5).
